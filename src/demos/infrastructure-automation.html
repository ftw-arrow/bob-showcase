Project Overview: Mike Jump Host

This demonstration showcases Bob's ability to automate complex infrastructure deployments. The project deployed a complete jump host solution in Azure with automated DNS registration, Kubernetes services, and enterprise-grade security.

What Was Automated

1. Azure VM Deployment

  • Ubuntu 22.04 LTS virtual machine
  • Automated SSH key configuration
  • Security group rules for SSH and RDP
  • Public IP with DNS label

2. DNS Configuration

  • Automatic GoDaddy DNS record creation
  • CNAME records for mikejmp.rcoace.com
  • Service-specific subdomains (ssh, rdp)
  • Validation and propagation checks

3. Kubernetes Services

  • LoadBalancer services for SSH and RDP
  • Automatic external IP assignment
  • Service health monitoring
  • Port forwarding configuration

Deployment Process

Step 1: VM Creation

$ az vm create \ --resource-group rg-mikejmp \ --name mikejmp \ --image Ubuntu2204 \ --size Standard_B2s \ --admin-username azureuser \ --generate-ssh-keys \ --public-ip-address-dns-name mikejmp ✓ VM created successfully ✓ Public IP: 20.12.34.56 ✓ DNS: mikejmp.eastus2.cloudapp.azure.com

Step 2: DNS Registration

$ ./create-dns-records.ps1 Creating DNS records in GoDaddy... ✓ CNAME: mikejmp.rcoace.com → mikejmp.eastus2.cloudapp.azure.com ✓ CNAME: ssh.mikejmp.rcoace.com → mikejmp.rcoace.com ✓ CNAME: rdp.mikejmp.rcoace.com → mikejmp.rcoace.com DNS propagation: Complete (45 seconds)

Step 3: Kubernetes Services

$ kubectl apply -f mikejmp-ssh-service.yaml $ kubectl apply -f mikejmp-rdp-service.yaml service/mikejmp-ssh created service/mikejmp-rdp created Waiting for LoadBalancer IPs... ✓ SSH Service: 20.12.34.57:22 ✓ RDP Service: 20.12.34.58:3389
